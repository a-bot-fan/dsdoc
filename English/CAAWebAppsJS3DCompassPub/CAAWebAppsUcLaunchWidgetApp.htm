What You Will Learn With This Use Case

In this article you will learn how the widget can inform the Compass to launch a widget application, and how the widget can, by return, be informed when the Compass has processed the request.

This use case illustrates the use of DS/i3DXCompassServices/i3DXCompassPubSub module.

The CAAi3DXLaunchWidgetApp Use Case

CAAi3DXLaunchWidgetApp is a use case of the CAAi3DXCompassUI.edu framework.

What Does CAAi3DXLaunchWidgetApp Do

CAAi3DXLaunchWidgetApp is a widget launching an application (*) for which the identifier is entered in the Preference menu:

Fig.1 Application Identifier

Once the "Save" Button is pushed, the widget launches the request, and is waiting for the Compass answer as specified in the widget body. But since this step can be extremelly fast, you could not have time to face the following trace:

Fig.2 Waiting for Response

When the Compass has sent its response, the widget body displays it:

The list of possible responses, and their meaning, is available in the technical article [2].

(*)About the application choice

The widget of the use case does not check the type of the application to launch. You can enter any kind of application: widget/web or native. MAP-CTRONGSAG is the identifier of the third app CAA Hello World use case [3] in our environment of development.

The end user can delete the widget (last icon), insert it in a dashboard (the middle icon), or refresh it (first icon).

Step-by-Step

There are four main steps:

  • Creating a JSON Object for the Request
  • Subscribing for a Compass Response
  • Publishing the Launch Request
  • Processing Compass Response

The first three are consecutive (in the same method), while the last is a callback function.

Creating a JSON Object for the Request

First you create a JSON object with the following keys/values: 

var params = {
    widgetId : widget.id ,
    appId    : widget.getValue('appid'),
};

If appId (the application identifier) is the only mandatory key, we strongly recommend to also pass the widgetId key. The identifier of the widget is the id property of the widget class. It is usefull when filtering the Compass response.

The params object is used in the Publishing the Launch Request step.

Subscribing for a Compass Response

Before requesting the launch of the application, the widget subscribes to the Compass response:

i3DXCompassPubSub.subscribe('launchAppCallback',MyWidget.onCallBackApp) ;

i3DXCompassPubSub is the argument of the anonymous function - see AMD Concept [5] - exported by the DS/i3DXCompassServices/i3DXCompassPubSub module.

The Compass sends the launchAppCallback event when the request is over. This is why we make the widget subscribe to this event. The second argument is the callback function, invoked each time the Compass processes a launch request.

Publishing the Launch Request

Finally the widget informs the Compass (indirectly) to launch the application:

i3DXCompassPubSub.publish('launchApp',	params);

i3DXCompassPubSub is the argument of the anonymous function - see AMD Concept [5] - exported by the DS/i3DXCompassServices/i3DXCompassPubSub module.

The Compass listens to all events with launchApp as name. This is why we make the widget subscribe to this event. params is the JSON object specifying the request defined in the first step.

Processing Compass Response

The callback method is called each time the Compass has finished to process a request. For the current widget, as for all widgets. So it is important to check, if it is the response for the current one:

onCallBackApp: function( launchData ) {
    if ( (launchData.widgetId === widget.id) &&
         (launchData.appId === widget.getValue('appid')) ) {
        var text="The status for launching " ;
        text += launchData.appId ;
        text += " is:" + launchData.response;
        widget.body.innerHTML=text;	
    }					                       
}

launchData, the callback function input, is a JSON object containing the identifier of the widget which has launched the request, the application requested, and the launch status.

If you compare the widget identifier and the application identifier contained by the JSON object against the current widget values a safe filtering can be performed.

In Short

This use case shows how a widget can launch a widget application, and be informed when the Compass has proceeded the widget's request.
