How to Use MyXCAD Data Type
Use Case

Abstract

This Use case illustrates MyX-CAD functionality creation and opening of CAA assembly and CAA component as an example with sample CAA type/data. It is important to understand this is sample data type in this application, user can also creates his own data type or create his own application and replace this application.

Basic Objective

The basic objective of this Use case is that, the user will be able to understand that how the basic menu is appearing in the MyX-CAD. And also able to use the data model\type given in the application. We have here two data type which are Component a standalone data type and Assembly Structure.

Basic Menu

Launch MyX-CAD from MyX-CAD launcher as define in the article [1]. After Launching under File menu we will get the basic menu structure as shown in [Fig.1].

Fig.1: Basic Menu

Next step when we click on the New Button we will get the File Data type. As show in the [Fig.2]

Fig.2: New Menu

Next we have to select the file data type which we want to create the file from New Menu.

As explained above we have two sample data type:

  • Component creation
  • Assembly creation

Creation of File Data Type

Component Creation

The component is basic entity of data type which is standalone entity. We have to select the Component under the New menu as shown in Fig.3.

Fig.3: Component Creation

After Selecting the Component we get following window for creation of component. in the following example shown in Fig.4 we are creating a single point co-ordinates.

Fig.4: Single Point Component Creation

After entering the appropriate value for the Point co-ordinates, height for the volume and By giving proper relation between the line element we get the one component and it is ready for save. The Fig.5 shows the Example for Component File, Which Contain 3 points and 3 line made between the same point which forms a triangle shape and the height is given for this shape which will give the volume for the component.

Fig.5: Component Creation
Saving Component File

After creation of component file we need to save it in local directory. so after clicking on save button in same New Menu bar we get the windows as shown in Fig.6. Save the file with some name in desired directory with correct format type for ex. MyCmp.CAAcmp.

Fig.6: Save Component

Assembly Creation

The Assembly structure which can contain one or more component/assembly in it. Assembly data type contain the other file in it. For creation of assembly structure we have to select the Assembly in New Menu as shown in the Fig.7.

Fig.7: Assembly Menu

After Selecting the Assembly we get following window [Fig.8] for creation of Assembly. in this we have add the Component or Assembly file to build the assembly structure.

Fig.8: Assembly Creation

Here we have added the two component in order to create the assembly structure Fig.9. We can also add assembly file in this assembly structure.

Fig.9: Assembly Example
Saving Assembly File

After creation of Assembly file we need to save it in local directory. so after cliking on save button in same New Menu bar we get the windows as shown in Fig.10. Save the file with some name in desired directory with correct format type for ex. MyAssm.CAAasm.

Fig.10: Save Assembly

We have this data file in local directory now for chekin to remote server ENOVIA V6 refer article[2].

In Short

We have seen the sample CAD application ( MyX-CAD App.) and its User Interfaces with its data type component and assembly, and how we can create this Sample Data type and save them.
