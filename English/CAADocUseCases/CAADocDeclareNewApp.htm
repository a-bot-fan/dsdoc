Declaring Third Party Native App in 3DCompass
Technical Article

Abstract

This article explains how to declare a new Native Client User App as a Third Party App in the 3DEXPERIENCE Compass. It is accomplished using the Platform Management console as it is described in user assistance documentation. The present article details the expected information to give through the user interface. The last section explains how you can do it by web service consumption.

Please note that only a 3DEXPERIENCE platform administator can manage Third Party Apps.

Create Third Party App Interactively

The first information to enter are the same whatever the Third Party App type (Native/Widget/Web).

  • Short Name: This is the name of the app that appears in the Compass.
  • Long Name (optional): This is the name displayed in the tooltip that appears when the end user passes hover the app in the Compass.
  • Compass Quadrant: Drop-down list, select the quadrant that best fits your app experience.
  • Type: Drop-down list, select Native.

Then the information only valid for a native application.

  • Exe

    Let's 3DEXPERIENCE.exe for a workbench, otherwise enter the name of the batch (bat only), or the name of the executable. The .bat or .exe file must be installed within the runtime view (/code/bin or /code/command directory) of your Rich Client installation.

  • ID

    The meaning is different from the precedent field.

    • 3DEXPERIENCE.exe

      It is the name of the app description class that implements the add-in CATIxxxConfiguration interface deriving from CATIWorkbench, and declared in the .dico file, such as CAAAfrGeoCreationWkb in the Creating an App use case

    • Custom application

      The custom application is launched thanks to CATSTART.exe. This executable comes with the 3DEXPERIENCE Launcher installation. There are several arguments passed by CATSTART to your own custom application. Among them there is this field value. The value is pre-fixed with -workbench key word. 

      custom.exe or custom.bat  .... -workbench ID_Value ...

      Without ID value, the workbench key word does not exist as input argument.

  • Custom Error Message

    While the 3DEXPERIENCE Launcher is not installed, instead "Your app is not installed", you can have your own message.

Managing Third Party App with Web Services

The 3DEXPERIENCE platform Compass is managed by the 3DCompass service. There is one web service to add third party app:

HTTP Request infovalue
Root URL3DCompass service URL
Application path/resources/AppsMngt/app/custom
VERBPOST
BODY typeFORM
Headers
  • Content-type=application/x-www-form-urlencoded;charset=UTF-8
  • Accept=application/json
BODY contents

The mandatory parameters are:

parameter namevalue
displaynameShort name in UI
platformsOnPremise for on premises solution, or the platform instance value for cloud solution

The optional parameters are:

parameter namevalue
iconThe complete path of the application icon
typeWeb|Native|Widget
quadrantNorth|West|South|East
launchinfosThe web URL|The ID field value|The widget URL
tooltipLong name in UI
visibilityPublic|Private
exeThe custom application name. Only for native applications
errorThe custom error message. Only for native applications

If you use the batch explained in the Testing 3DSpace Web Services article, with the following lines in the input file you can add a native third party app.

10=POST./resources/AppsMngt/app/custom 
10_FORM=quadrant=West&launchinfos=ID_BATCH&type=Native&displayname=My Batch For Test&platforms=...&exe=batch.bat

You only have to adapt the platforms parameter value to your environment.

In Short

This article describes how to declare a new Native Client App through the interactive application, and using web service.
