What's New in 3DEXPERIENCE R2016x REST Web Services
Technical Article

Abstract

This article presents REST Web Services and associated documentations that are added since 3DEXPERIENCE R2016x GA, compared with the previous release. Some enhancements or maintenance reasons may also require changes regarding API usage or methodologies. This article lists them, and points to the corresponding documentation.

3DSwym

The R2016x-GA brings some changes in the services exposed by 3DSwym.

The first one is about the CSRF token. The name of the header to pass to the https request is now X-DS-SWYM-CSRFTOKEN instead X-SWYM. Please refer to the CSRF Token section of the pointed article for more details.

3Swym-FD01

The changes are for Delete Answer and Delete Answer Comment web services. They both used to return: 

 {"result":23,
  "nb_result":true,
  ...}

Now it is:

  {"result":{"id":23,"moderation_status":"deleted"},
   "nb_result":true,
  ...}

The content of the result key is no more an integer (the id of the deleted element), but a JSON object containing the integer and a status.

3Swym-FD02

The changed occured in FD01 are now valid for Delete Post Comment and Delete Media Comment web services.

For Edit Media and Edit Post, the attribute category_label has been removed from the POST body. It has never been taken into account.

The hidden key has been deprecated. Its value is meaningless since it is always 0. For the same reason the delete key has been deprecated when it is a status deletion of a resource. The delete key for the _acl key is still valid.
