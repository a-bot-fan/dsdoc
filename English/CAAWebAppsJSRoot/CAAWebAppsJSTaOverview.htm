3DEXPERIENCE Applications
Technical Article

Abstract

The 3DEXPERIENCE platform consists of a bunch of applications accessible to the end user through the Compass. At first, this article presents the three kinds of application: Native, Web and Widget. Then it introduces the sub nodes of the 'Web and Widget Apps Javascript" node meanly leading to the construction of a Widget App.

Native, Web and Widgets Apps

The 3DEXPERIENCE platform consists of a bunch of applications accessible to the end user through the Compass:

These applications are launched through three kinds of client:

  • Native

    Such applications are named "Native Apps". They were named "workbenches". Each aplication is launched in a unique rich client. You launch a first native app, then a second, the first is replaced by the second in the same rich client.

  • Web

    Such applications are named "Web Apps". They were named "centrals". Each application is launched in a unique web page. You launch a first web app, then a second, the first is replaced by the second in the same web page.

  • Dashboard

    Such applications are named "Widget Apps". Each application is "instantiated" in tab pages of the 3DDashboard. You can have several widgets at the same time.

There is still an exception for 3DSwym: you click the icon, you launch a specific web page. You drag and drop the icon, you launch a widget in a dashboard

Widget Apps Development

The purpose of the "Web and Widget Apps JavaScript" node is mainly to explain how to develop a widget app. The explanations to develop a widget app have been split in three main sections:

  • Widget Creation

    You will learn the fundamentals for a widget based on UWA Protocol: its identity (metadata), its behavior (events), its contents (preferences), its running.

  • Widget Interactions

    You will learn the fundamentals to interface a widget with the 3DEXPERIENCE platform User Experience components as the 6WTagger or how widgets can communicate between them.

  • Widget and Data Access

    You will learn how a widget can access any services of the 3DEXPERIENCE platform, as external ones.
