Installing and Launching a Widget App Use Case
Technical Article

Abstract

This article is intended to show you how to install and launch a widget app use case in a 3DEXPERIENCE platform.

Foreword

A widget app use case is an html page usually requiring other files such as JavaScript or resources files. Everything is set up in one or several mweb modules of frameworks available on the API CD-ROMs. All the stuff must be installed on a server accessible from the 3DDashboard of your 3DEXPERIENCE platform.

The following folder names are used in this article.

Folder Name Meaning
InstallRootFolder The pathname of the folder into which the Dassault Systèmes' solutions and products run time and API CD-ROMs were unloaded.
MyRootFolder The pathname of a root folder on your server.
CAAFramework.edu The education framework that contains the use case module.
CAAUseCase.mweb The use case module, or one of the modules that make up the use case.

Installing the Widget App

For each mweb module making up the use case:

  1. Copy the InstallRootFolder\CAADoc\CAAxxx.edu\CAAyyy.mweb\src folder in the MyRootFolder folder on your server.
  2. Rename the src folder as CAAyyy, the mweb module name (without the mweb extension).

Just below an example with the modules of the CAAWebAppsJS.edu framework making up the Defining and Using AMD Modules use case [1]

On the left the folders as delivered on the media, and on the right the result of the copy, and the rename. The src folder under CAAWebAppsOperationApp.mweb (on the left) is copied/pasted as CAAWebAppsOperationApp under MyRootFolder on the right. The same operation is repeated for the two other mweb modules.

In this example, the widget app html file (CAAWebAppsOperationApp.html) is located in the CAAWebAppsOperationApp folder. This and the two others folders, contains resources files in the assets folder, and javascript files just under or in sub-folders such as basic and complex.

Launching the Widget App

As explained in the Widget 3D Dashboard Integration [2] article, there are two ways to launch a widget app in the 3DEXPERIENCE platform:

  1. Through the Compass, if the widget is previously declared as Third Party.
  2. Through the Run your App widget app (East quadrant).

The last one is fastier and does not require to be platform's administrator, but cannot be used if the application invokes a 3DEXPERIENCE platform service.

In both cases, an URL of the widget app is requested. It is as follows:

$ServiceURL/MyRootFolder/CAAyyy[/...]/widgetApp.html

Where ServiceURL is the URL of your server.

The widget app html file can be just under the app module, or in a sub folder. In the CAAWebAppsOperationApp case, it is $ServiceURL/MyRootFolder/CAAWebAppsOperationApp/CAAWebAppsOperationApp.html.
