Launching MyX-PDM Use Case

MyX-PDM functionalities are integrated in MyX-CAD application in the form of addins. To use this addins we have to launch MyX-CAD application and load all this addins. Launching MyX-CAD application is explained in How to Launch MyX-CAD[1].

Once MyX-CAD is started successfully, we can launch MyX-PDM using MyX-PDM launcher.

  • Create runnable jar for packages under MyXPDMLauncher.mj module,it should be named as MyXPDMLauncher.jar.
    Fig.1: Package Structure
  • Right click on package name MyXPDMLauncher, select Run as from menu and choose Java Application. This is shown in Fig.2
    Fig.2: Run Application
  • In case of successful execution of above command following window Fig.3 appears:
    Fig.3: MyX-PDM Launcher
  • "Install path " : is path of .jar files, $Java Project workspace$\bin\win_b64.
  • These files contains the information about bus adapter implementation used in integrating MyX-PDM with the 3DEXPERIENCE platform.
  • When we click on "Launch MyX-PDM", information about ongoing processes will be displayed in MyX-PDM information area. Fig.4 shows the sucessful execution of My-XPDM launcher. In this case,login to MyX-PDM operation is performed at MyX-CAD side.
  • Refer to How to Use MyX-PDM [2] for details about this application.
    Fig.4: MyX-PDM Application
  • In MyX-PDM, code for integration of MyX-PDM with 3DEXPERIENCE platform, is packaged in CAAVIXPDMIntegration.edu framework. This integration includes send and receive functionalities.

Framework packaging is shown in Fig.5. CAAXPDMSampleBusAdapter.mj module combines different components packaged in implementation of sample bus adapter for MyX-PDM.

Fig.5: Package Structure

Note: In order to use Bus Adapters, customer can use packages from CAAVIXPDMIntegration.edu framework.

To use X-PDM add-in functionalities (e.g. Send to 3DEXPERIENCE platform, Receive from 3DEXPERIENCE platform), we have to start few applications which are deployed on application server.

Application which are deployed on Application server(e.g. Tomee Plus) can be launched by starting the application server. i.e. execute startup.bat present at $ApplicationServer$\bin. If it successfully executes then it shows message as: INFO : Server startup in --- ms.(Miliseconds mentioned in ms ).
