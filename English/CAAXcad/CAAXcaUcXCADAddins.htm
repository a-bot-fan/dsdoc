X_CAD Add-ins
Use Case

Abstract

The article describes the flow and basic steps for store the data type to ENOVIA V6 server.

Basic Objective

The basic objective of this functionality is that, the user will be able to manage the files of MyX-CAD Application in ENOVIA V6 server.

Launch ENOVIA V6 in MyXCAD

Launch the MyX-CAD application [1] and create the data model type according to the steps given in article [2]. After creation of data type in MyXCAD Application we need to store this files in the database. Here we have selected the ENOVIA V6 Server.

After executing the ENOVIA V6 Button following window will appear. Which contains the menu [Fig.1].

  • Connect to ENOVIA V6.
  • Check in to ENOVIA V6.
  • Checkout from ENOVIA V6.
  • Disconnect of ENOVIA V6.
Fig.1: ENOVIA V6 Menu

Next step when we select Connect to ENOVIA V6, Then Login window for ENOVIA V6 will appear [Fig.2]. This login is to connect the our application with the remote server.

Before Login to ENOVIA V6 check the following details.

  • Start the ENOVIA Server.
  • Launch IEF Client on client server.
  • Save the file Locally.

Now provide proper details such as User ID and password which are compulsory for login to server.

We can also give the vault name in ENOVIA V6. After giving proper details just select login Tab for login.

Fig.2: Login Window for EV6

Checkin to ENOVIA V6

Here we have to checkin the file(Assembly or component) to server i.e. to store the file in to the server. before checkin to ENOVIA V6, file should be locally saved. after executing this command following window will appear[Fig.3].

Fig.3: Checkin Window

Here we can see that before check-in it ask for the creation of version of file in server, we can just tick on the checkbox if we want to create the version. and we can easily check the file to server be selecting 'check-in' button.

The steps involved for an Integration to check-in files Assembly scenario into ENOVIA are explained in the article [3].

Checkout from ENOVIA V6

This command is required if want to checkout the file from ENOVIA V6 server which is already saved in the server data base.

After selecting the tab 'Checkout from ENOVIA V6' in menu bar we will get the following window.

We can see in the [Fig.4] that we have search the file name 'assembly' which is already present in the data base. we will get the result in the search box, we have to select the file in the search box which we want to checkout and execute the 'Checkout' button.

Fig.4: Checkout Window

The steps involved for an Integration to check-out files Assembly scenario into ENOVIA are explained in the article[4].

After Checkout the popup window will appear where we need to provide the path where we want to store the checkout file on local machine [Fig.5].

Fig.5: Saving Path for Checkout

Disconnect from ENOVIA V6

After execution of all operation just click on the 'Disconnect of ENOVIA V6' for safely Logout from the server and break linking between the server and Client MyXCAD Application [Fig.6].

Fig.6: Disconnect from ENOVIA

In Short

We have learn that how we can save the file to remote server through checkin command and how to checkout (retrive) the file from the server.
