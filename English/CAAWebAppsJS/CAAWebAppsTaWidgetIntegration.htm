Widget 3D Dashboard Integration
Technical Article

Abstract

In a first glance the "Compass Your Business" [1] article presents the three different ways to integrate a web application in the 3D Dashboard. For each case it valids, or not, some main feature capabilities. This article goes further with technical explanations.

Third Party Apps

It is the way for production purpose to integrate a Widget App into the 3DEXPERIENCE platform.

Principle

"Third Party Apps" is a widget of the 3DEXPERIENCE platform only accessible by its administrators. It enables them to "push" a widget into a platform instance. The widget access can be granted for all people of the platform, or part of them. The granted people have access to the widget through a quadrant of the 3DCompass.

Just above you can see the "CAA My Test" widget integrated into the East Quadrant. Once available the end user can drag and drop the widget into a dashboard. Just below the "Third Party Apps" widget:

On the left, the information about the widget (short and long name, icon, type 'widget', URL , a 3DCompass quadrant), and on the right a preview.

  • The short name
    • This name is the first part of the widget title.
  • The URL:
    • It must point to a web application based on UWA protocol. See the definition in the referenced technical article [2].
    • The server hosting the widget app must be accessible from the 3DDashboard server.

3DDashboard Integration

The widget html page (hosted on any server) is "injected" into a widget (green one) hosted by the 3DDashboard. The Widget Platform engine loads this transformed widget in the 3DEXPERIENCE browser into a "trusted" 3DDashboard domain. The transformed widget is an iframe (so isolated from the dashboard page) where the DS UWA is imbedded. The principle of the injection is detailed in the iframe Widget section.

Capabilities

  • The widget (trusted) is granted to access the 3DEXPERIENCE servers (3DSwym, 3DSpace, ...).
  • The widget can reach external servers (non 3DEXPERIENCE servers) thanks to proxified API [3] .
  • The widget can interface with the 3DEXPERIENCE user experience components as the 6WTagger.
  • The widget takes beneficit of the 3DDashboard capabilities: communication between widgets, automatic deletion, ... .
  • The widget has the DS UWA behavior, the UWA version available in the 3DEXPERIENCE.

Run Your App

It is the quick way, but with limited capabilities, to integrate a Widget App into the 3DEXPERIENCE platform.

Principle

"Run Your App" is a widget of the 3DEXPERIENCE platform. It enables any end-user to integrate a Widget App into a dashboard.

About the URL:

  • It must point to a web application based on UWA protocol. See the definition in the referenced technical article [2].
  • The server hosting the widget app must be accessible from the 3DDashboard server.

3DDashboard Integration

The widget html page (hosted on any server) is "injected" into a widget (green one) hosted by the 3DDashboard. The Widget Platform engine loads this transformed widget in the 3DEXPERIENCE browser into a "untrusted" 3DDashboard domain. The transformed widget is an iframe (so isolated from the dashboard page) where the DS UWA is imbedded. The principle of the injection is detailed in the iframe Widget section.

Capabilities

  • The widget (untrusted) is not granted to access the 3DEXPERIENCE servers (3DSwym, 3DSpace, ...).
  • The widget can reach external servers (non 3DEXPERIENCE servers) thanks to proxified API [3] .
  • The widget can interface with the 3DEXPERIENCE user experience components as the 6WTagger.
  • The widget takes beneficit of the 3DDashboard capabilities: communication between widgets, automatic deletion, ... .
  • The widget has the DS UWA behavior, the UWA version available in the 3DEXPERIENCE.

Web Page Reader

It is the way to integrate an existing stand alone web application into a 3DEXPERIENCE dashboard.

Principle

"Web Page Reader" is a widget of the 3DEXPERIENCE platform. It enables any end-user to insert a stand-alone web app:

The web application - integrated in the Web Page Reader widget - runs as in stand-alone.

About the URL:

  • It can be any stand-alone web application.
  • The server hosting the widget app must be accessible from the 3DDashboard server.

3DDashboard Integration

The Web Page Reader widget, as any dashboard widgets, is also isolated from the other widgets by an iframe. This iframe contains another iframe pointing to the web application on the hosting server.

Capabilities

The web app enclosed in the Web Page Reader runs as stand-alone, but without 3DEXPERIENCE capabilities:

  • The web app cannot access the 3DEXPERIENCE servers (3DSwym, 3DSpace, ...).
  • The web app cannot interface with the 3DEXPERIENCE user experience components as the 6WTagger.
  • The web app cannot take beneficit of the 3DDashboard capabilities: communication between widgets, automatic deletion, ... .

But the Web Page Reader widget instance can be shared as other widgets in a dashboard.

In Short

Three ways:

  • Third App, for a widget with full 3DEXPERIENCE capabilities,
  • Run Your App, for a widget without 3DEXPERIENCE server access,
  • Web page Reader, for an existing stand-alone web application with only the capability to be shared for another dashboard.
