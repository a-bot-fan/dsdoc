List Current User Communities

Role

Returns a list of the communities associated with the current user. Default filter returns all communities the user is member of.

Request

 Method  Path 
GET api/community/listmycommunities

Parameters

The GET parameters (see GET Method) are:

 Name   Require   Value   Description 
creation_granted_for no A string. Possibles values are:
  • post: return only communities where the user can create post.
  • media: return only communities where the user can create media.
  • iquestion: return only communities where the user can create question.
  • ideation: return only communities where the user can create ideation.
  • wikitree: return only communities where the user can create wiki.
 Filter result by user insertion rights policy.
limit no A number between 1 and 128. Default value if not indicated is 10. The maximum number of elements returned by the list.
page no A number greater than one. Default value if not indicated is 1. Optional parameter to be used with "limit". For paginated result (see Paginated Result).

Response

The response is a standard 3DSwym JSON (see JSON as Output) where the key result is an array. Each element of the array is a JSON composed as follows:

 Key   Description   Value   Applicability 
access Current user access to the community. A number. Possible values are:
  • 1: contributor ; user can read content in the community depending on moderation rules and write comments.
  • 2: author ; same rights than contributor ; user can additionally create content in the community such as Posts.
  • 3: co-owner ; same rights than author ; user can additionally accept users in the community and change its settings.
  • 4: current user is super user of the platform, a.k.a. platform administrator ; he/she has full access to the community.
 yes

delete no

extranet_access Visibility of the community for contractors of the platform. A number. Possible values are:
  • 0: private ; community is returned in search results but cannot be accessed by partners.
  • 1: public ; everybody on the platform can see it.
  • 4: secret ; community is not returned in search results and cannot be accessed by partners.
yes

  • 0: the community cannot be seen by partners.
  • 1: the community can be seen by partners.
yes

ideation_is_enabled Status of ideation on the community. A boolean:
  • 1: ideation is enabled on the community.
  • 0: ideation is not enabled on the community.
