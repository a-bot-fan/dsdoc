List Community Members

Role

Returns a list of the users that a are members of the community. [quel est le filtre? qu'est-ce qu'un member]

Request

 Method  Path 
POST api/community/listmembers

Parameters

None.

POST Body

The value is a JSON composed as follows:

 Name   Require   Value   Description 
params yes The value is a JSON composed as follows:
 Name   Require   Value   Description 
community_id yes A positive number. Identifier of the community to list members.
limit no A number between 1 and 128. Default value if not indicated is 10. The maximum number of elements returned by the list.
page no A number greater than one. Default value if not indicated is 1. Optional parameter to be used with "limit". For paginated result (see Paginated Result).
Container for all post parameters.
request_id yes 0 For internal use.

Response

The response is a standard 3DSwym JSON (see JSON as Output) where the key result is an array. Each element of the array is a JSON composed as follows:

 Key   Description   Value   Applicability 
LastActivity User last activity date (latest contribution) in the community. A date (see Date Format). yes
activity no
city City part of the user address. A string. An empty string if information is not shared. yes
company Company where the user works. A string. An empty string if information is not shared. yes
country Country part of the user address; English name. A string. yes
ds_user no
email E-mail of the user. A e-mail string. Ex: john.doe@ds.com. An empty string if information is not shared. yes
first_name First name. A string. An empty string if information is not shared. yes
hash_key Crypted identifier. A hash key. yes
id no
job_title User job title. Example: CEO. A string. An empty string if information is not shared. yes
lastVisit User last visit date in the community. A date (see Date Format). yes
last_name Last name. A string. yes
login no
month_nb_answer Number of answers to iQuestions published by the user in the last month in the community. A positive number. yes
month_nb_comment Number of comments published by the user on blog posts, iQuestions, media in the community during last month. A positive number. yes
month_nb_media Number of media uploaded by the user in the community during last month. A positive number. yes
month_nb_post Number of blog posts in the community created by the user in the last month. A positive number. yes
month_nb_question Number of iQuestions the user has asked in the last month in the community. A positive number. yes
nb_answer Number of answers to iQuestions published by the user in the community. A positive number. yes
nb_comment Number of comments published by the user on blog posts, iQuestions, media in the community. A positive number. yes
nb_media Number of media uploaded by the user in the community. A positive number. yes
nb_post Number of blog posts in the community created by the user. A positive number. yes
nb_question Number of iQuestions the user has asked in the community. A positive number. yes
nickname yes
picture_update yes
rank_points Total number of activity points in the community. Statistics are based on the number of contributions in the community. A positive number. yes
relation Role in the community of the current user. A number. Possible values are:
  • 1: contributor.
  • 2: author.
  • 3: owner.
 yes
user_id no
user_picture_fingerprint no
user_tenant_status Indicate if the user is an employee (0) or a partner (1). A number. yes

ut_invitation_email Email identifying a user with invitation pending. An e-mail string. yes
ut_state State of the user on the platform A number. Possible values are:
  • 0: invitation pending.
