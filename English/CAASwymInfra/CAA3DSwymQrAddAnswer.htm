i Add Answer

Add Answer

Role

Adds an answer to an existing iQuestion.

Request

 Method  Path 
POST api/answer/add

Parameters

None.

POST Body

The value is a JSON composed as follows:

 Name   Require   Value   Description 
params yes The value is a JSON composed as follows:
 Name   Require   Value   Description 
answer yes A string. Format is HTML-like (see Format HTML-like). Text of the answer.
qna_question_id yes A positive number. Identifier of the iQuestion where the answer will be added.
Container for all POST parameters.
request_id yes 0 For internal use.

Response

The response is a standard 3DSwym JSON (see JSON as Output) where the key result is itself a JSON composed as follows:

 Key   Description   Value   Applicability 
answer Answer message. A string. Format is HTML-like (see Format HTML-like). yes
answer_id Answer identifier. A positive number. yes
can_validate Validation status for the current user. A boolean:
  • 0: cannot validate an answer.
  • 1: can validate one.
 yes
community_id Community identifier onto which the iQuestion has been published. A positive number. yes
question_id iQuestion identifier. A positive number. yes
