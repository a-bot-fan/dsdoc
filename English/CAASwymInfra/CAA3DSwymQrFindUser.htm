Find User

Role

Finds a user based on his username or hash key. Returns information on the found user or nothing if no information has been found. Information depends on the user information sharing policy. The user is identified either by a login name or a hash key value.

Request

 Method  Path 
GET api/user/find

Parameters

The GET parameters (see GET Method) are:

 Name   Require   Value   Description 
hash_key no A crypted identifier. Crypted identifier of the user to get, if username is not available.
login no A string. Username of the user to get.

Response

The response is a standard 3DSwym JSON (see JSON as Output) where the key result is itself a JSON composed as follows:
