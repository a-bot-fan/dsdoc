List Media Comments

Role

Returns a list of the comments belonging to a media. Returns also deleted comments (see moderation_status in response).

Request

 Method  Path 
GET api/media/listcomments

Parameters

The GET parameters (see GET Method) are:

 Name   Require   Value   Description 
limit no A number between 1 and 128. Default value if not indicated is 10. The maximum number of elements returned by the list.
media_id yes A positive number. Identifier of the media to list comments from.
order yes A string. Possible values are:
  • asc: return results in ascending order (preferred display value).
  • desc: return results in descending order.
 Order for the comments inside the list.
order_by yes A string. Possible values are:
  • crdate: order on creation date (preferred display value).
  • update: order on update date.
 Ordering result for the comments inside the list.
page no A number greater than one. Default value if not indicated is 1. Optional parameter to be used with "limit". For paginated result (see Paginated Result).

Response

The response is a standard 3DSwym JSON (see JSON as Output) where the key result is an array. Each element of the array is a JSON composed as follows:

 Key   Description   Value   Applicability 
_acl Access Control List (ACL) per content type for current user.

The value is a JSON composed as follows:

 Key   Description   Value   Applicability 
delete Delete ACL for media comment.
