Edit Answer Comment

Role

Edit an existing comment of an answer to an iQuestion. Returns the edited comment.

Request

 Method  Path 
POST api/answer/editcomment

Parameters

None.

POST Body

The value is a JSON composed as follows:

 Name   Require   Value   Description 
params yes The value is a JSON composed as follows:
 Name   Require   Value   Description 
comment yes A string. Format is HTML-like (see Format HTML-like). Text of the comment.
id yes A positive number Identifier of the comment to edit.
Container for all POST parameters.
request_id yes 0 For internal use.

Response

The response is a standard 3DSwym JSON (see JSON as Output) where the key result is itself a JSON composed as follows:

 Key   Description   Value   Applicability 
comment Text of the comment. A string. Format is HTML-like (see Format HTML-like). yes
id Comment identifier A positive number. yes
moderation_date Moderation date A date (see Date Format). yes
moderation_status Status of the edition of the comment. A string. Possible values are:
  • updated: updated by a community owner or a platform administrator.
  • updated_by_author: updated by the author of the content.
 yes
moderation_userid User identifier of a person that has edited the comment (moderator). If the comment owner has edited or no deletion has occurred on the comment, null identifier (0) is returned. A positive number. yes
