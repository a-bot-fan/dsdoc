Launching an Automation Use Case
 
This article gives you the hints to execute an Automation use case.

This article is divided in three steps:

  1. Create the Macro Library
  2. Insert or Create the Macro Source
  3. Search Type Specificity

  1. Create the Macro Library

    1. Either press ALT + F8, or from the Tools section of the action bar, click Macros .

      The Macros dialog opens.

    2. Click Macro libraries.

      The Macro libraries dialog opens.

    3. In the Library type list, select PLM VBA projects.

    4. Click Create new library.

      The Macro library VBA dialog opens.

    5. In the Title box, type the name you want to assign to the macro library, or keep the default name.

    6. In the Description box, possibly type a description for the macro library to create.

    7. Click OK.

    8. Click Close in the Macro libraries dialog box.

    9. Click Create in the Macros dialog box.

      The Create a new macro dialog box opens.

    10. In the Macro name box, type a name for the module to be created in the VBA macro library, or keep the default name.

    11. Click OK.

      The Create a new macro dialog box closes and the created module in selected in the Macros dialog box

    12. Click Edit.

      The Microsoft Visual Basic for Applications window opens.

    13. If the macro code editor of the created macro is not yet opened, in the Project pane, expand the macro library up to the created module.

    14. Right click this module, and click View Code.

  2. Insert or Create the Macro Source

    1. Either type ALT + F8, or from the Tools section of the action bar, click Macros .

      The Macros dialog opens.

    2. In the Current macro library or document list, select a macro library.

    3. Click Edit.

      The Microsoft Visual Basic for Applications window opens.

    4. Possibly create a new module: In the Project pane, right click your just created macro library, point to Insert, and click Module.

    5. Right click this module, and click View Code.

    Usually every article describing a use case points to an html file containing the whole source code of its macro. You only have to copy/paste the source's lines in the macro code editor, and to execute it.

  3. Search Type Specificity

    When the use case contains a database search, i.e. uses the SearchService service object, it requires a PLM type as input. It can be given by the end user through a dialog box, or it can be hard coded in the macro. In this latter case, you may have to adapt the source code to your own PLM types. The use cases requiring a possible change are identified, and they explain how to do the change.
