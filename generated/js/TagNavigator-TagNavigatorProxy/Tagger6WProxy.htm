addEvent(eventName, eventCallback, scope)

Sets callback functions, used for setting callbacks for filtering events

Parameters:
Name Type Argument Default Description
eventName

Name of the event you want to set a callback on. It can be

  • 'onFilterChange' if the proxy filteringMode is "FilteringOnServer"
  • 'onFilterSubjectsChange' if the proxy filteringMode is "WithFilteringServices"
eventCallback

Function called when the specified event is triggered. Its signature is depending on event:

  • filterChangeListener if the event is 'onFilterChange'
  • filterSubjectsChangeListener if the event is 'onFilterSubjectsChange'
scope <optional>
 undefined

Scope of the callback
Inherited From:
  • addEvent
Returns:

null

Example
     this.taggerProxy.addEvent('onFilterChange', eventCallback, this);
     this.eventCallback = function (filter) {
         // filter example "OnFilterChange":
         //{
         //    "allfilters": {
         //        "ds6w:when/ds6w:starts": [
         //            {
         //                "object": "2013",
         //                "type": "date"
         //            }
         //        ]
         //    }
         //}
         // filter example "OnFilterSubjectsChange":
         //{
         //   "filteredSubjectList": [
         //        'subject-1-uri',
         //        'subject-2-uri'
         //    ]
         //}
     };

addSubjects(tagsdata) → {}

Adds 6WTags into the 6WTagger, use this method if you need to display more selectable subjects and you summaryData does not change.

Parameters:
Name Type Description
tagsdata

Object json that represents the association between subjects and their 6WTags.

Properties
Name Type Description
subject-uri .<>

The URI of the subject. For each subject you have to provide an array of {predicate/object}

Properties
Name Type Argument Description
type <optional>

Type of the "object". Can be either "integer","string" or "date".
sixw

Hierarchical path of the predicates URIs, from root 6W to used predicate
object

6WTag object URI or literal. If summaryData.type is "date", the date must be of the following format : "yyyy/mm/dd".
dispValue <optional>

The object name displayed in the 6WTagger. If it is not present, summaryData.object value is used instead. (This attribute is not taken into account if the object type is "date") A tag which "type" attribute is not set is considered as a literal, that is to say a string.
Returns:

The current Tagger6WProxy instance

Type
Example
     var tagsdata =  {
         'subject-1-uri':[
             {'object':"Alice",'sixw':"ds6w:who/ds6w:responsible", 'dispValue':"Alice"},
             {'object': "2011/05/30", 'sixw': "ds6w:when/ds6w:start", 'dispValue': "WILL NOT BE DISPLAYED", 'type': "date"},
             {'object':"Post",'sixw':"ds6w:what/ds6w:type", 'dispValue':"Post"}
         ],
         'subject-2-uri':[
             {'object':"Thomas",'sixw':"ds6w:who/ds6w:responsible", 'dispValue':"Thomas"},
             {'object':"Post",'sixw':"ds6w:what/ds6w:type", 'dispValue':"Post"}
         ]
     };
     this.taggerProxy.addSubjects(tagsdata);

die()

Deletes the 6WTaggerProxy. First, it removes the 6WTags from the 6WTagger, and then deletes the connection from the 6WTagger. This proxy does not exist anymore for the 6WTagger. Usually this method is called when the widget is destroyed. You cannot use the proxy once the die method is called, all methods will fire an exception.

Inherited From:
  • die
Returns:

null

Focus on subject(s), the 6WTagger will display only tags of the focused subject(s). To enable tag edition mode of the tagger: the widget has to be maximized, the proxy active, and subject(s) has to be focused.
