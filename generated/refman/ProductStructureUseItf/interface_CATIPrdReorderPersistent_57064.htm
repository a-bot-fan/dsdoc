Methods


o GetChildrenInPersistedOrder
public virtual GetChildrenInPersistedOrder( iProductReorderableType,
oListOrderedChildren)
Retrieves the children of the current component in their persisted order (database order or last order set with PersistChildrenOrder).
Parameters:
iReorderableType
Type of the children to retrieve.
oListOrderedChildren
List of retrieved components of the provided type (empty if the type is not managed). The list must be empty when the method is called. Each element of the list must be released by the caller after use.
Returns:
E_INVALIDARG : The list was not empty when the method was called. or The specified type is not managed. The list is returned empty. E_FAIL : An internal error occurred. S_OK : The children were successfully retrieved.
o GetReorderableType
public virtual GetReorderableType( ipComponent,
oProductReorderableType)
Retrieves the type of a component. Can be useful, but no impact on order.
Returns:
- S_OK if the component type can be retrieved - E_INVALIDARG if the component is of unknown type or is null (type is set to PRTUnmanagedType) - E_FAIL if an unexpected error occurs.
o PersistChildrenOrder
public virtual PersistChildrenOrder( iListOrderedComponents,
iListModifiableComponents)
Sets a new persistent order for the children of the current component. This method does not propagate the modifications. The new order can be retrieved through GetChildrenInPersistedOrder, even before propagation.
Parameters:
iListOrderedComponents
List of components to order. Components can only be of one CoreType.
iListModifiableComponents
List of modifiable components to order.
Returns:
E_INVALIDARG : Elements of different CoreTypes were passed in any of the list. E_FAIL : An internal error occurred. S_OK : The new order was set.
