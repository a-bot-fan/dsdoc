Usage: you can freely reimplement this interface.

interface CATIPLMCheckBeforePropagate

Interface to add a check step before propagating.

Role: This interface allows a check to be added in the propagate command. When this interface is at least once implemented, a push button is added in the Propagate panel. This push button allows being propagated data to be checked. While the end user does not push this check button, the OK button is disabled and propagation is forbidden. OK becomes enabled when all the checks (i.e all the method) are successful (return S_OK)

This interface can be multi-implemented as a data extension of a PLMCheckBeforePropagate_XXXXXcomponent. The XXX string is your choice while PLMCheckBeforePropagate_ is imposed.

The implementation call order is not guaranted

Methods


o CheckBeforePropagate
public virtual CheckBeforePropagate( oMsg)
Performs the check.
Role: This method is called when the Check button of the Propagate panel is pushed. The availability of the OK button that is in charge of Propagation launching depends on the returned code of this method.
When implenting this method, any business logic can be coded to check the consistency of data before propagation is launched.
Parameters:
oMsg
[out] Message displayed as an easy warning to the user.
Returns:
  • S_OK if Check is OK. OK button of Propagate panel is allowed to be available and an easy warning is displayed.
  • S_FALSE if Check fails. OK button of Propagate panel stays unavailable and an easy warning is displayed.
  • E_FAIL otherwise. No warning is displayed
o SetCheckDisplayName
public virtual SetCheckDisplayName( oName)
Provides the displayed name of the Check.
Role: This method provides the check name that is displayed:
  • in the short help of the check button.
  • in the easy warning indicating the result of the check.
Parameters:
oName
[out] Check name displayed in the check return easy warning and in short help of the check button.
Returns:
  • S_OK if check name correctly valuated.
  • E_FAIL otherwise. Easy warning and short help do not content check names.
o SetLabel
public virtual SetLabel( oLabel)
Provides the label of the Check button.
Role: This method provides the label of the check button which is added in the Propagate panel.
Parameters:
oLabel
[out] Label of the check button to be added in the Propagate panel. When several checks are implemented, the SetLabel result is not taken into account and the label of the check button is "Check"
Returns:
  • S_OK if Label correctly valuated.
  • E_FAIL otherwise. The label of the check button is "Check"
